## Overview

The type layout module can be built using a combination of the `TypeLayout` and [`TitleBar`](#references) components.

The `TypeLayout` accepts `TypeLayoutBody`, `TypeLayoutCell`, and `TypeLayoutRow` components as `children`.

## Component API

<ArgsTable />

Additional props passed into `TypeLayout`, `TypeLayoutBody`, `TypeLayoutCell`, and `TypeLayoutRow` will be forwarded along to their underlying containers.

### `size`

This prop specifies which size the `TypeLayout` should be rendered at. Valid values are `xs`, `sm`, `md`, and `lg`. If no size is specified, it renders as `md`.

<Canvas withSource="none">
  <Default size="sm" />
</Canvas>

```jsx
<TypeLayout size="sm" />
```

## `border`

This prop specifies whether a border should be rendered.

<Canvas withSource="none">
  <Default border />
</Canvas>

```jsx
<TypeLayout border />
```

### `className`

The `className` prop passed into either `TypeLayout`, `TypeLayoutBody`, `TypeLayoutCell`, and `TypeLayoutRow`, will be forwarded along to their underlying containers. This is useful for specifying a custom class name for layout.

```jsx
<TypeLayout className="class-name" />
```
