## ButtonCluster

The button cluster module group secondary actions that can be taken on a whole page or component. Icons are used to help describe, at a glance, the action that can be taken.

### Overview

The button cluster module can be built using a combination of the `ButtonCluster` and [`Button`](#references) components. The `ButtonCluster` accepts a list of `Button` components as `children`.

### Grid placement

Button cluster modules span across 4 columns. Following the [narrow grid](#references) implementation, the layout module container hangs 16 pixels in the left margin. The narrow grid position allows headings and copy outside of containers to align with the labels within the button cluster module.

```jsx
<Column lg={4}>
  <ButtonCluster>
    <Button kind="ghost">Button 1</Button>

    <Button kind="ghost">Button 2</Button>
  </ButtonCluster>
</Column>
```

### Layout considerations

Occassionally a [spacing token](#references) will need to be included to offset the button cluster module to ensure it top-aligns with the adjacent layout container. If a top spacing token is necessary, reference the layout module that sits directly above it as the starting point for placement.

### Anatomy

1. **Ghost buttons:** Ghost buttons stack directly below each other
2. **Divider lines:** Divider lines create structurality and provide separation between the ghost buttons
3. **Bottom spacer:** The 16-pixel spacer `$layout-01` below the last button provides the starting point for the layout module positioned directly underneath it
